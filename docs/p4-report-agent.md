# P4 — 보고서 에이전트 요건

> 이 문서는 사용자가 직접 요건을 기재하는 공간입니다.
> AI는 이 문서를 읽고 구현 방향을 결정합니다.

## 현재 구현 상태

- `STAGE2_SYSTEM_PROMPT`: 배경/이슈/검토/계획 구조, 개조식, 금감원 문체 적용
- `RunConfig` 패턴으로 MCP 도구 비활성화, 확정 결론을 고정 입력으로 사용
- CLI에서 Stage 1 확정 후 자동/직접입력/건너뜀 선택 흐름 구현

## 보고서 형식 상세

<!-- 금감원 보고서 구체적인 형식 요건을 기술하세요 -->

I. 보고서 구조

1. 현황 (왜 이 보고서를 작성하게 되었는지)
2. 이슈 (어떤 것이 의사결정이 필요한 이슈사항인지, 필요하다면 대안을 포함)
3. 검토의견 (작성자가 판단하는 처리방향)
4. 향후 계획 (해당 처리방향이 수락된다면, 향후 구체적 action plan, 타 부서 협의, 금융위 협의, 전금법 개정 건의 등)
<!-- 예: 각 섹션 분량 제한, 개조식 규칙, 각주 형식 등 -->

II. 보고서 분량
기본적으로 모든 분량은 붙임을 제외하고 hwp 1~2페이지 이내여야 한다.
보고서는 개조식이다.
1 / □ / ○ / - / · 순으로 Level이 내려간다.
각 개조식 문단의 단위는 한 문장으로 구성된다.
주석이 필요한 경우 *을 사용한다.

# 간단한 예시

[금융결제원 뱅크페이서비스의 PG업 등록 필요여부 검토]
1. 배경
□ 금융위는 '26.4.19.자로 우리원 앞 금융결제원의 전자금융거래법 상 직불업 등록 필요여부 검토를 요구
 ○ 금융결제원은 스스로의 지위는 전금법 제정취지상 전자금융보조업자*라는 입장
   * 전금법 제정 당시 결제중계시스템 운영자(금융결제원)을 전자금융보조업자로 명시

2. 이슈
□ 현재 오픈뱅킹, 펌뱅킹 등 계좌이체방식 결제서비스 제공 주체에 대한 필요 라이센스 해석이 상이
 ○ 금융위는 24년 계좌이체방식 결제서비스를 직불수단으로 해석한 바 있음
 ○ 금융위, 금결원은 24년 하위사업자 연계한 오픈뱅킹 이용기관에 대해서는 직불업 등록을 요구
  - 그러나 하위사업자 및 직접 서비스를 제공하는 이용기관에 대해서는 일관성 부족

3. 검토의견
□ 행정지도를 통한 직불업 요구 필요
 ㅇ 오픈뱅킹, 펌뱅킹 등 경로와 무관하게 서비스를 제공하는 사업자에 대해 직불업 요구하는 것이 타당

4. 향후 계획
□ 금융위 앞 본 검토방향에 대한 의견요청(즉시)
□ CPC발송을 통한 영향분석(즉시)
□ 수단이 아닌 행위에 기반한 전금법 라이센스체계 정비방안 건의(장기)
