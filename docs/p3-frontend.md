## 역할 및 목표

<!-- 프론트엔드가 수행해야 하는 핵심 역할을 기술하세요 -->
<!-- 예: 이슈 입력, 에이전트 진행상황 실시간 확인, confirm/feedback 인터페이스 등 -->

사용자는 콘솔기반이 아니라 프론트엔드 기반으로 이 프로젝트 센티널과 상호작용한다.
백엔드는 계속 항상 서빙할 필요는 없다.
Stage1, Stage2, Stage3 각각에 대해 요청하면 그때 작업을 개시한다.
각각은 독립적이고, 각 스테이지 간 만약 필요하다면 정보전달은 db를 경유한다.
현재까지 구현된 콘솔에서 입출력되는 것을 프론트엔드(브라우저)를 통해 상호작용한다.

## 화면 구성 (UI 레이아웃)

<!-- 주요 화면과 각 화면의 구성 요소를 기술하세요 -->
최상단에는 tab 3개가 있다. stage1, stage2, stage3
stage1.
이슈를 input받아서 검토결론을 return하는 것이므로 텍스트박스들로 구성

stage2.
방법1. db에 적재된 stage1 결과물 중에 선택하거나, 
방법2. 검토결론을 수기로 사람이 써서 제시할수도 있음.
두 방법 중 선택에 따라 요청되면, p4-report-agent.md에 기재된 보고서 작성요령에 따라 보고서 form의 문자열로 작성한다.
방법1로 이전 stage 결과물을 선택한 경우, 선택 직후 원문 전체가 텍스트박스에 축약 없이 표시되어야 하며 사용자는 그 자리에서 직접 수정할 수 있어야 한다.
별도의 "추가 개발 요청" 입력칸을 두지 말고, 사용자가 텍스트박스에서 수정한 결과 자체를 다음 stage 입력으로 전달한다.

stage3.
stage2 확정 보고서 또는 기존 stage3 편집본을 선택해 시작할 수 있다.
이 경우에도 선택한 원문 전체가 텍스트박스에 축약 없이 표시되어야 하며, 사용자가 직접 편집한 텍스트를 stage3 편집 초안 생성 입력으로 사용한다.
